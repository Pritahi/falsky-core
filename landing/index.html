SEC. 01 // DRILL LOG

HOW DETECTION WORKS

01

Falsky uses five independent signal components to calculate a trust score (0-100) for every test. Each component is weighted, and flaky categories trigger additional penalties.

01

Bayesian Pass Rate

Beta-Binomial conjugate prior (α=2, β=1). Handles small sample sizes better than naive pass/total. Primary signal at 55% weight.

Component Weight: 55%
02

Recent Trend Analysis

Compares pass rate of last 5 runs vs overall. Detects improving or degrading tests before they become critical.

Component Weight: 20%
03

Duration Stability

Z-score analysis on test durations. |z| > 2.0 = timing anomaly. Coefficient of variation > 0.5 = suspicious.

Component Weight: 10%
04

Error Pattern Consistency

Normalized error message pattern analysis. Different errors = more suspicious. Same error = real bug.

Component Weight: 10%
05

Flaky Penalty

When a flaky category is detected, an additional penalty is applied: timing (−8), order (−12), shared state (−15), non-deterministic (−20).

Penalty: −8 to −20

SEC. 05 // INSTALLATION

DEPLOY THE SENSOR

Add Falsky to your GitHub Actions workflow. One YAML block. Zero configuration. Start detecting flaky tests in under 2 minutes.

▶ Start Monitoring Free
FREE TIER · 1 REPO · 100 RUNS/MONTH · NO CREDIT CARD
ci.yml
# .github/workflows/ci.yml
name: CI with Falsky
on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm test -- --reporter junit
      - uses: Pritahi/falsky-action@v1
        with:
          api_key: ${{ secrets.FALSKY_API_KEY }}

# → Trust scores posted to PR
# → Flaky tests classified
# → Seismic alerts on failure
